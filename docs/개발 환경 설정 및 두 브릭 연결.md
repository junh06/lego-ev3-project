# 1. 서론
컵 인식과 잡기, 들기, 비우기 등의 복합적인 행동을 수행하는 ev3를 만드는 것이 목적이다. 이를 위해서는 정밀한 모터 제어와 두 개의 브릭간의 긴밀한 통신이 필수적이었다. 기존의 레고 마인드스톰 전용 소프트웨어(블록 코딩)로는 이러한 복잡한 로직과 통신을 구현하는데 한계가 있다고 판단하여, Python 기반의 텍스트 코딩 환경을 구축하기로 결정하였다.

# 2. 개발 환경 구축
# 2-1. ev3dev 운영체제와 VS Code
레고 공식 사이트의 EV3 MicroPython Getting Started Guide를 참고하여 다음과 같은 순서로 개발 환경을 구축하였다. 먼저 공식 사이트에서 EV3 MicroPython micro SD card image를 다운로드한 후, Etcher 프로그램을 이용해 microSD card에 플래싱하였다. 플래싱 된 card를 ev3브릭에 삽입하고 전원을 켜서 ev3dev OS로 부팅하였다. vs code 확장 프로그램에서 LEGO MINDSTORMS EV3 MicroPython을 설치하고 새 프로젝트를 생성했다. PC와 EV3를 미니usb케이블로 연결한 후 EV3DEV DEVICE BROWSER에서 브릭을 연결하였다.
# 2-2. 트러블 슈팅
문제 상황: 초기 환경 설정 중 마지막 단게에서 vs code와 브릭을 연결하려 했으나, 성공 표시(녹색) 대신 연결 실패(빨간색) 오류가 발생했다.

원인 분석: 윈도우 환경에서 EV3를 네트워크 장치(NDIS)로 인식하는 과정에서 IP 할당 및 드라이버 설정이 자동으로 이루어지지 않았다.

문제 해결

1. PC 설정: [제어판] - [네트워크 연결]에서 현재 사용 중인 인터넷(Wi-Fi)의 속성으로 들어가 '인터넷 연결 공유'를 활성화했다. 이때 홈 네트워킹 연결 대상을 EV3가 연결된 어댑터(Remote NDIS)로 지정했다.

2. EV3 설정: 브릭 메뉴의 Wireless and Networks -> All Network connection -> Wired -> IPv4 설정에서 'Load Windows defaults'를 선택하여 윈도우가 할당한 IP를 받아오도록 설정했다.

3. 이후 다시 연결을 시도하자 vs code에서 브릭이 정상적으로 인식(녹색 표시)되었다.
# 3. 두 브릭의 통신 시스템
제작한 레고는 상단(Top)과 하단(Bottom) 두 개의 브릭으로 나뉘어 있으며, 이 둘은 하나의 몸처럼 유기적으로 움직여야 한다. 이를 위해 두 브릭 간의 통신 시스템을 설계하였다.
# 3-1. 초기 시도: Bluetooth
처음에는 선꼬임을 방지하고자 블루투스를 이용한 무선 통신을 시도하였다. Top브릭이 클라이언트, Bottom 브릭이 서버가 되어 메시지를 주고받는 방식이다.

문제점: 페어링 과정에서 Host is down 메시지가 뜨며 연결이 거부되거나, 연결되더라도 3초 만에 끊어지는 등 통신 안정성이 매우 낮아 프로젝트 진행이 불가능했다.
# 3-2. 최종 해결: USB Daisy-Chain 기반 유선 소켓 통신
시스템의 안정성이 최우선이라는 판단하에 무선을 포기하고 미니USB 케이블을 이용한 유선 소켓 통신으로 수정하였다.

물리적 연결: Top브릭의 USB 포트와 Bottom 브릭의 PC 포트를 USB 케이블로 직접 연결(Daisy-Chain)하였다.

네트워크 설정

Top Brick (Master/Client) <=(USB Cable)=> Bottom Brick(Slave/Server)

Top Brick 설정

TARGET_HOST = '169.254.187.149' - Bottom IP

TARGET_PORT = 9999

Bottom Brick 설정

HOST = '0.0.0.0'

PORT = 9999

# 4. 결론 및 성과
유선 소켓 통신으로 전환한 결과, 데이터 유실이나 끊김 없이 안정적인 송수신이 가능해졌다. 이를 통해 두 개의 독립된 브릭이 마치 하나의 뇌를 가진 것처럼 즉각적으로 반응하는 실시간 동기화를 구현할 수 있었다. 이번 경험은 로봇 제어 시스템에서 환경에 맞는 통신 프로토콜을 선택하는 것이 얼마나 중요한지를 깨닫는 계기가 되었다.
